```python
# websocket_manager.py
from fastapi import WebSocket
from typing import Dict, Set

class ConnectionManager:
    def __init__(self):
        self.active_connections: Dict[int, Set[WebSocket]] = {}

    async def connect(self, user_id: int, ws: WebSocket):
        await ws.accept()
        self.active_connections.setdefault(user_id, set()).add(ws)

    async def send_personal_message(self, user_id: int, message: dict):
        for ws in self.active_connections.get(user_id, set()):
            await ws.send_json(message)

manager = ConnectionManager()

# main.py
@app.websocket("/ws/{user_id}")
async def websocket_endpoint(user_id: int, ws: WebSocket):
    await manager.connect(user_id, ws)
    # listen for events and broadcast
```

**Неделя 2: Frontend WebSocket**


## Заключение

**EasyCyberPro** имеет сильное уникальное преимущество (тренерская система + СНГ-фокус), но критически отстаёт в ключевых областях: визуализация турнирных сеток, интеграция с играми, античит и масштабируемость.

Предложенный план из 8 фаз (~35 недель) покрывает все выявленные недостатки и выводит платформу на уровень паритета с FACEIT/Battlefy по ключевым функциям, сохраняя при этом уникальное УТП.

**Главный приоритет:** Фаза 1 (Bracket UI) — без визуализации турниров платформа не может полноценно проводить соревнования.
